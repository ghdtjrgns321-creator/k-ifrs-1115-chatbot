# K-IFRS 1115 회계감사 AI 어시스턴트

> **K-IFRS 제1115호(고객과의 계약에서 생기는 수익)** 전문 AI 도구
> 회계감사인이 객관적 팩트를 먼저 확인하고, AI의 구조화된 가이드로 판단을 내릴 수 있도록 설계

---

## 왜 이 프로젝트가 필요한가

범용 AI(ChatGPT, Gemini)에게 회계 질문을 하면 **환각(Hallucination)** — 근거 없이 그럴듯한 답변을 생성하는 현상 — 이 발생합니다.
회계감사에서 환각은 **2종오류**(실재하는 회계 쟁점 누락)로 직결되며, 부실감사·제재·투자자 피해로 이어집니다.

이 시스템은 AI에게 자유로운 추론을 허용하지 않습니다.
**PydanticAI**로 출력 양식을 코드 수준에서 강제하고, **Decision Trees**(26개 토픽별 체크리스트 + 결론 가이드)로 AI가 답변할 수 있는 범위를 제한합니다.

| 항목 | 일반 ChatGPT | 이 시스템 |
|------|-------------|----------|
| 답변 근거 | 학습 데이터에서 자유 추론 | DB에 저장된 기준서 원문만 사용 |
| 근거 추적 | 불가능 (출처 불명) | 문단 번호 단위로 추적 가능 |
| 환각 방지 | 없음 | PydanticAI 5단계 구조적 방어 |
| 결론 구조 | 자유형 텍스트 | Decision Trees 체크리스트 + 분기별 결론 가이드 |
| 정보 부족 시 | 근거 없이 단정 | 조건부 결론 (Case 1/2)으로 분기 제시 |
| 검색 | 없음 | 2계층 하이브리드 검색 + Reranker 2차 검증 |
| 모델 | 단일 모델 | 질문 유형별 최적 모델 자동 선택 (듀얼트랙) |
| 품질 검증 | 없음 | 53개 골든셋 자동화 회귀 테스트 |

## 사용자 경험 흐름

### 흐름 1: 토픽별 문서 열람

```
관심 토픽 버튼 클릭 (예: "변동대가", "본인 vs 대리인")
  │
  ▼
4개 탭으로 관련 문서 즉시 열람
  ├─ [본문·BC]     기준서 원문 + 결론도출근거
  ├─ [적용사례]    IE 적용사례 (실무 예시)
  ├─ [질의회신]    QNA 101건 중 해당 토픽 관련
  └─ [감리사례]    FSS·KICPA 감리지적사례
```

30개 토픽별로 관련 문서가 사전 큐레이션되어 있어 **검색할 필요 없이 바로 열람** 가능합니다.

### 흐름 2: 질문 → AI 답변 + 근거 Split View

```
┌──────────────────────┬──────────────────────┐
│  [좌측] 근거 문서     │  [우측] AI 답변       │
│                      │                      │
│  ▸ 본문·적용지침      │  [결론]              │
│  ▸ 적용사례 (IE)     │   Case 1: ~이면 본인 │
│  ▸ 질의회신 (QNA)    │   Case 2: ~이면 대리인│
│  ▸ 감리사례          │  [논리적 근거]       │
│  ▸ 참고 추가 문서    │   문단 B37~B38 ...   │
│                      │  + 유사 감리사례 경고 │
│                      │  + 꼬리질문          │
└──────────────────────┴──────────────────────┘
```

AI 답변과 **AI가 가져온 근거 문단**을 한 화면에서 나란히 확인할 수 있습니다.

### 흐름 3: 멀티턴 대화

거래 상황 질문에서 AI는 한 번에 결론을 내리지 않습니다. **체크리스트 형태의 꼬리질문**으로 핵심 판단 요소를 하나씩 확인합니다.

```
사용자: "A사가 B사를 통해 소프트웨어를 판매합니다. 본인인가요?"

AI 1턴: "판단에 필요한 핵심 요소를 확인해주세요:
  □ B사가 재화를 통제한 후 고객에게 이전하나요?
  □ B사가 가격결정권을 보유하나요?
  □ B사가 재고위험을 부담하나요?

  현재 정보로는:
  [Case 1] B사가 통제권 보유 → 본인 (총액 인식)
  [Case 2] A사가 통제권 보유 → 대리인 (순액 인식)"

사용자: "B사는 가격 결정권이 없고, 재고는 A사에 반품 가능합니다"

AI 2턴: "[결론] B사는 대리인에 해당할 가능성이 높습니다.
  [근거] 문단 B37~B38에 따르면..."
```

## 핵심 기술

### Decision Trees + 환각 방지 5단계 레이어

26개 토픽별로 **Gemini·Claude·NotebookLM 3개 AI를 교차검증**하여 작성한 판단 체계입니다.
각 Decision Tree는 **7개 섹션**(라우팅 키워드 → 체크리스트 → 결론 가이드 → 선례 → 감리 경고 → 계산 공식 → 핵심 판단 요소)으로 구성됩니다.

| # | 방어 레이어 | 역할 |
|---|------------|------|
| 1 | **Decision Trees** | 26개 토픽별 체크리스트 + 결론 가이드. AI가 답변할 수 있는 범위를 정의 |
| 2 | **PydanticAI 구조화된 출력** | `인용 문단` + `선택 분기`를 반드시 채워야 답변 가능 |
| 3 | **result_validator 자동 재시도** | 인용이 비어 있으면 반려 → AI 재생성 (최대 2회) |
| 4 | **reasoning_guard** | 결론 가이드에 없는 분기를 AI가 임의 생성하는 것을 차단 |
| 5 | **선례 직접 주입** | 큐레이션된 적용사례·질의회신·감리사례 원문을 AI에 직접 전달 |

### 2계층 하이브리드 검색

| 계층 | 방식 | 특징 |
|------|------|------|
| **Tier 1** | 핀포인트 검색 | Decision Tree에 배정된 문서 ID로 MongoDB 직접 조회. 누락 위험 0% |
| **Tier 2** | 하이브리드 검색 | 벡터 검색(의미 유사도) + BM25(키워드 정확 매칭) + RRF 융합 |
| **Reranker** | 2차 검증 | Cohere Cross-encoder로 재평가. 핀포인트 문서는 보호 |

### 듀얼트랙 모델 라우팅

7개 모델을 218회 호출하여 비교한 결과, **회계 추론과 산술 정확도가 단일 모델로 양립 불가**함을 확인했습니다.

| 모델 | 역할 | 강점 |
|------|------|------|
| **Gemini Flash** (thinking=medium) | 일반·상황 질문 | 회계 추론 품질 1위 (0.81) |
| **gpt-4.1-mini** | 계산 질문 | 산술 정확도 100% |

### 5노드 파이프라인

```
사용자 질문
  │
  ▼
[Analyze] gpt-4.1-mini — 질문 분석·라우팅 (10개 판단 항목)
  │
  ▼
[Retrieve] 2계층 검색 — 핀포인트 + 하이브리드 (병렬)
  │
  ▼
[Rerank] Cohere Cross-encoder — 비즈니스 가중치 + 핀포인트 보호
  │
  ▼
[Generate] 듀얼트랙 — Gemini(판단) / gpt-4.1-mini(계산)
  │
  ▼
[Format] 감리사례 경고 + 꼬리질문 + 인용 정리
```

## 데이터 파이프라인

6종 약 1,575건의 데이터를 14단계 파이프라인으로 수집·가공합니다.

| 데이터 유형 | 건수 | 출처 | 용도 |
|------------|------|------|------|
| 기준서 본문·적용지침·BC | ~1,300 청크 | kifrs.com | 기준서 원문 검색·인용 |
| 적용사례 (IE) | ~150 문단 | kifrs.com | 실무 적용 예시 |
| 질의회신 (QNA) | 101건 | kifrs.com | 실무 해석 선례 |
| 감리지적사례 | 18건 | FSS·KICPA | 규제 리스크 경고 |
| KAI 교육자료 | 5건 | 한국회계기준원 | AI 답변 품질 보강 |
| 토픽 큐레이션 | 30개 토픽 | AI 교차검증 | 체크리스트 + 결론 가이드 |

**청킹 전략**: 기준서 본문은 문단 단위 + 카테고리별 가중치, QNA·감리사례는 3단 분할(Q/A/S) + PDR(Parent Document Retrieval)

## 설계 한계 (의도된 Trade-off)

| 대응 가능 | 대응 불가능 |
|----------|------------|
| 체크리스트 Q&A, 조건 추가, 결론 확정 | 타 기준서 교차 (1116호 리스 등) |
| 개념 질문 ("변동대가란?") | Gray Area (메타버스, DeFi) |
| 계산 질문 ("진행률 계산해줘") | 자유형 사례 요청 ("다른 회사는?") |

> AI의 자유도를 제한하는 것이 곧 환각을 방지하는 핵심 메커니즘입니다.
> 이 한계는 버그가 아닌 **의도된 설계**입니다.

## 프로젝트 핵심 교훈

> **"AI의 품질은 AI 모델이 아니라 데이터와 구조에서 결정된다"**

- 검색 적중률 **+40%p** — 모델 교체가 아니라 **데이터 파싱 결함 수정**
- 산술 정답률 **+67%p** — 모델 업그레이드가 아니라 **질문 유형별 모델 분리**
- 꼬리질문 일관성 **60%→100%** — 프롬프트 추가가 아니라 **추론 강도를 낮추는 것**
- 환각 방지 **100%** — AI 기술이 아니라 **AI 교차검증 큐레이션(Decision Trees)**
