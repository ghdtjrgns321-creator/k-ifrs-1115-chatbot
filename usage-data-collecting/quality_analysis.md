# 챗봇 품질 분석

> 실사용 5건 기준 | 2026-03-16

## 결론부터

**챗봇이 잘 작동하고 있다.** 5건 중 4건이 A등급, 사용자 피드백도 👍.
다만 거래상황 질문은 25초로 느리고, 범위 밖 질문의 채점 방식에 허점이 있다.

---

## 잘 되고 있는 것

**개념 질문은 완벽에 가깝다.**

"변동대가란?", "계약자산 vs 수취채권 차이?" 같은 개념 질문에서:
- 12초 내 답변 완료
- 기준서 문단 4~5개 정확 인용
- 질의회신(QNA)까지 자동으로 찾아서 인용

예시 — "반품권이 있는 판매에서 수익 인식 시점은?"
> 문단 56, B21, B23, B25 + QNA-SSI-35567 인용 → 사용자 👍 피드백

**거래상황 질문의 판단 품질이 좋다.**

"SW 라이선스 + 3년 업데이트" 질문에서:
- 토픽 3개 정확 매칭 (라이선싱, 수행의무 식별, 기간 vs 시점)
- 결론 분기 3개 모두 제시 (결합 수행의무 / 접근권 / 사용권)
- 기준서 6건 인용 (문단 27, B54, B56, B58, B61, 사례 55)

---

## 고쳐야 할 것

**1. 거래상황 질문이 25초 걸린다**

개념 질문은 12초인데, 거래상황은 25초. 체크리스트 분석 + 분기 결론 생성 때문.
다만 이미 ADR-28에서 후속 턴은 thinking=low로 최적화했고, 첫 턴은 품질 우선이라 허용 범위.

→ **지금은 놔둬도 된다.** 품질(인용 6건, 분기 3개)이 속도보다 중요.

**2. 범위 밖 거절을 "저품질"로 잘못 채점한다**

"테스트 질문" → 범위 밖 거절 → 인용 0건 → 인용 커버리지 0점 → 종합 0.60.
거절이 정상 동작인데 저품질로 찍힌다.

→ **채점 로직에서 거절 응답을 제외해야 한다.**
