## 4. 개선 필요 영역

### 🔴 우선 (Partial 케이스)

1. **TEST-0 (본인 vs 대리인)**: Run 3에서 clarify 분기를 거치지 않고 단정적 결론 도출. analyze 단계에서 "위탁판매 가능성"이 있는 경우 clarify를 강제하는 로직 보강 필요

2. **TEST-2 (완결된 복합 사례)**: 불확실성이 이미 해소되었다는 정보가 주어졌음에도 3회 중 2회 불필요 체크리스트 질문 생성. analyze 단계에서 "불확실성 해소 여부"를 감지하는 로직 추가 고려

3. **SECTION-7 (진행률 측정)**: "유의적 관여" 분기가 체크리스트에 누락된 것으로 추정. `matched_topics`에 해당 분기 조건 추가 필요

4. **SECTION-8 (유의적 금융요소)**: timeout 에러 — 프로덕션에서 재시도 로직 필요

### 🟡 관찰 사항

- **리트리버 정확도**: 핵심 문단(B34~B38, B56~B58, 91~93 등) 매칭률 우수. 다만 B77(위탁약정 징후), IE 사례 문서는 직접 검색되기보다 LLM 인용에 의존하는 경향
- **에러율**: 1/78 = 1.3% — 인프라 수준 허용 범위이나 프로덕션에서는 retry 권장
- **응답 속도**: 평균 23.1초로 사용자 체감상 양호 (20~30초 밴드)
