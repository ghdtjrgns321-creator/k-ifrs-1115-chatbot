## 4. 개선 필요 영역

### 🔴 우선 (Partial 케이스)

1. **B1 (진행률 계산)**: 3회 중 1회만 정답(85억). calc 라우팅이 안 돼서 Gemini가 처리 → 미설치 자재 제외 원칙을 알면서도 계산에 반영 안 함. **원인**: `_needs_calculation()` 3조건 중 "계산해줘" 같은 명령형 동사가 "구해줘"인데, is_situation=True + calc 라우팅이 일관적이지 않음

2. **B2 (변동대가 기댓값)**: 원칙 설명은 정확하나 실제 숫자 산출을 3회 중 1회만 수행. calc 라우팅 일관성 문제 — 금액 3개 조건을 충족하나 "산정해주세요"가 `_CALC_COMMAND`에 매칭되지 않음

3. **B3 (반품권 계산)**: 수익/환불부채는 일관 정확하나 매출원가(1,080)+회수권(120)을 명시적으로 분리 산출하지 못하는 경우 존재

4. **C2 (라이선싱 3턴)**: T2에서 접근권 결론이 나왔는데 T3에서 불필요한 로열티/MG 질문을 다시 함. **원인**: checked_items에 T2 Q&A가 누적되었지만 "결론 도달 후 추가 질문 불필요" 판단이 약함

5. **C3 (범위 밖)**: Run 1에서 "증분차입이자율" 키워드가 1115호의 "금융요소" 토픽과 매칭되어 범위 밖 판정 실패. **원인**: analyze_agent가 "증분차입이자율"을 1115호 금융요소와 혼동

### 🟡 관찰 사항

- **라우팅 정확도**: A1~A4 모두 Pass — generate/clarify/calc 3경로 분기가 정확
- **양방향 분기 안전장치**: C1(본인vs대리인)에서 3회 모두 TYPE 1 → TYPE 2 전환 정상 작동 (개선 2 효과 확인)
- **멀티턴 맥락 유지**: C1, C2 모두 이전 턴 정보가 누적되어 중복 질문 없음 (C2의 T3 문제는 불필요 질문이지 중복은 아님)
- **calc 라우팅 일관성**: A3(계산해주세요)은 3/3 성공, B1(구해줘)은 1/3만 calc 진입. `_CALC_COMMAND` 패턴 확장 또는 "산정해주세요" 추가 검토 필요
