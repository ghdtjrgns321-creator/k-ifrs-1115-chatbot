## 왜 다른가 — 일반 AI vs 이 시스템 <sub>[상세: PROJECT-OVERVIEW](FINAL-REPORT/1_PROJECT-OVERVIEW.md)</sub>

   └──────────────────────┴──────────────────────┘

## 핵심 UX — "근거 선행, AI 후행"

이 시스템은 AI에게 바로 질문하는 일반 챗봇과 다르다.
**사용자가 먼저 기준서 원문을 확인하고, AI는 그 근거 위에서만 답변한다.**

- **30개 토픽 네비게이션** — 토픽 선택 시 연관 토픽 버튼이 함께 표시. 예) "할인액의 배분" → `거래가격 배분`, `변동대가의 배분`, `계약변경`
- **4탭 즉시 열람** — 본문·BC / 적용사례(IE) / 질의회신 101건 / 감리사례 18건. AI 호출 전에 검색 없이 근거를 확인
- **탭별 큐레이션 요약** — 각 탭과 개별 항목에 한 줄 요약(summary/desc) 제공. 전문을 펼치기 전에 핵심을 먼저 파악
- **관련 조항 자동 연결** — 문단 81 → 관련: `문단 82` + 출처 경로(`본문 > 측정 > 할인액의 배분`). 기준서 논리 구조를 따라가며 탐색
- **Split View 동시 확인** — 좌측 근거(AI 인용 문단 볼드) + 우측 구조화 답변(Case 분기 + 감리사례 경고 + 꼬리질문)

## 핵심 기술 <sub>[상세: SEARCH-AND-AI](FINAL-REPORT/4_SEARCH-AND-AI.md)</sub>

### 환각 방지 5-Layer Pipeline <sub>[상세: DOMAIN-CURATION](FINAL-REPORT/3_DOMAIN-CURATION.md)</sub>

데이터 적재부터 최종 답변까지 전 과정을 통제하는 **확정적(Deterministic) 아키텍처**:

1. **도메인 데이터 적재** — 검증된 K-IFRS 데이터 6종 1,575건을 DB에 저장 (본문·적용사례·질의회신·감리사례·KAI교육·토픽큐레이션)
2. **핀포인트 + Reranker 검색** — Tier 1 핀포인트(Decision Tree 문서 ID로 직접 조회, 누락 0%) + Tier 2 하이브리드(벡터 + BM25 + RRF 융합) → Cohere Cross-encoder 재평가
3. **듀얼 LLM 라우팅** — 7개 모델 218회 A/B 비교 → Gemini Flash(회계추론 1위) + gpt-4.1-mini(산술 100%) 자동 선택
4. **Decision Tree 강제** — 토픽별 의사결정 트리로 AI 자의적 추론 차단. 정보 부족 시 임의 판단 대신 조건부 결론(Case 분기) 도출
5. **PydanticAI 구조화 출력** — '근거'와 '결론' 분리를 코드 수준에서 강제. result_validator + 자동 재시도

### 5노드 파이프라인

```
사용자 질문
  │
  ▼
[Analyze] gpt-4.1-mini — 질문 분석·라우팅 (10개 판단 항목)
  │
  ▼
[Retrieve] 2계층 검색 — 핀포인트 + 하이브리드 (병렬)
  │
  ▼
[Rerank] Cohere Cross-encoder — 비즈니스 가중치 + 핀포인트 보호
  │
  ▼
[Generate] 듀얼 LLM 라우팅 — Gemini(판단) / gpt-4.1-mini(계산)
  │
  ▼
[Format] 감리사례 경고 + 꼬리질문 + 인용 정리
```

## 설계 한계와 향후 방향

### 설계 한계 — 자유검색 제한

이 시스템은 AI 챗봇이라기보다 **규칙 기반 전문 검색 시스템**에 가깝다.
AI에게 자유로운 추론을 허용하면 더 다양한 질문에 답할 수 있지만,
**근거 없는 답변(환각)이 구조적으로 불가능한 현재의 보장**이 깨진다.

| 대응 가능                          | 대응 불가능                                        |
|-----------------------------------|----------------------------------------------------|
| 체크리스트 Q&A 응답                | 관련된 최신 뉴스 검색해줘                            |
| 추가 조건 제시 및 결론 확정         | 코인관련 새로운 거래유형인데 어떻게 처리해야해?        |
| 개념 설명 · 계산 질문              | 다른 회사는 어떻게 처리해?                           |

> **유연성은 범용 LLM에 맡기고, 무결성에 집중한다.**
> 본 시스템은 범용 LLM이 보장하지 못하는 '환각이 없는 확정적 근거 제시'를 위해 설계되었다.
> 대화의 자유도를 잃더라도 엄격한 제약을 유지하는 것이, 이 프로젝트의 최우선 가치이다.

### 향후 방향

- **타 기준서 확장** — K-IFRS 1116호(리스), 1109호(금융상품) 등 별도 Decision Trees 구축
- **BIG4 가이드 임베딩** — 삼일·삼정·안진·한영의 K-IFRS 1115 실무 가이드를 데이터에 추가
- **RAGAS 자동 평가** — Faithfulness·Context Precision 등 자동 정량 평가 프레임워크 구축
