# 교훈과 한계

> 40건의 디버깅 교훈과 33건의 의사결정 기록 중, 실무적으로 의미 있는 10가지 교훈과 이 시스템의 한계·발전 방향을 정리합니다.

## 1. 디버깅 

###  전처리 교훈 3가지

| 교훈 | 설명 |
|------|------|
| **PDF 전처리는 최후의 수단** | 이미 구조화된 데이터(API, HTML)를 먼저 조사. PDF에서 표와 서식을 정확히 추출하는 것은 극히 어려움 |
| **기능별 폴더 분리** | 14개 전처리 스크립트가 하나의 폴더에 있으면 의존 관계 파악이 어려움. 기능별 분리가 유지보수에 유리 |
| **문자 깨짐 방지를 단단히** | 크롤링 직후에 유니코드 정규화를 수행하지 않으면, 이후 모든 단계에서 예측 불가능한 버그 발생 |

### 주요 디버깅 교훈

## 2. 설계 한계 — 의도된 Trade-off

이 시스템의 한계는 **버그가 아닌 의도된 설계**입니다. AI의 자유도를 제한하는 것이 곧 환각을 방지하는 핵심 메커니즘이기 때문입니다.

### 대응 가능 vs 불가능

| 구분 | 유형 | 예시 |
|------|------|------|
| **대응 가능** | 체크리스트 Q&A | "네, 고객이 개별적으로 효익을 얻을 수 있습니다" |
| | 추가 조건 제시 | "계약 기간은 3년입니다" |
| | 결론 확정 요청 | "결론을 내려주세요" |
| | 개념·계산 질문 | "변동대가란?", "진행률 계산해줘" |
| **대응 불가능** | 타 기준서 교차 질문 | "리스(1116호)와 결합하면 어떻게 되나요?" |
| | Gray Area | "메타버스 가상자산 수익인식은?" |
| | 자유형 사례 요청 | "다른 회사는 어떻게 처리하나요?" |
| | 멀티턴 중 새 문서 요청 | "관련 사례 더 보여주세요" |

### 왜 이 한계를 유지하는가

이 시스템은 AI 챗봇이라기보다 **"규칙 기반 전문가 시스템"**에 가깝습니다.

감사 매뉴얼을 AI가 임의로 수정할 수 없듯이, Decision Trees에 정의된 체크리스트·결론 분기·선례만을 사용하여 답변합니다. AI에게 자유로운 추론을 허용하면 더 다양한 질문에 답할 수 있지만, 동시에 **근거 없는 답변(환각)이 구조적으로 불가능한 현재의 보장**이 깨집니다.

## 3. 향후 발전 방향

### 단기 (현재 아키텍처 내)

| 항목 | 내용 | 기대 효과 |
|------|------|----------|
| **듀얼 모드 (V2)** | 구조화된 답변(formed) 완료 후, "자유 질문 모드" 전환 버튼 제공. 자유 모드에서는 환각 방지 레이어가 약화됨을 안내 | 대응 가능 범위 확대 + 사용자 기대 관리 |
| **교차 질문 매칭**  | 하나의 질문이 2개 이상 기준서에 걸칠 때 동시 처리 | 복합 거래 질문 대응 |
| **Gray Area 방어** | 새로운 사업모델(DeFi, 메타버스) 관련 질문 감지 시, 기존의 구조화된 답변이 아님을 명시 후, Pydantic AI가 아닌 자유질문 LLM을 통한 답변 제시 | 범위 밖 질문의 사용자 경험 개선 |

### 중장기 (아키텍처 확장)

| 항목 | 내용 | 기대 효과 |
|------|------|----------|
| **BIG4 가이드 임베딩** | 삼일·삼정·안진·한영 회계법인의 K-IFRS 1115 실무 가이드를 데이터에 추가 | 실무 해석 범위 확장 |
| **RAGAS 자동 평가** | Faithfulness(근거 충실도), Context Precision(검색 정밀도) 등 자동 정량 평가 프레임워크 구축 | 개선 효과를 수치로 즉시 검증 |
| **타 기준서 확장** | K-IFRS 1116호(리스), 1109호(금융상품) 등 별도 Decision Trees 구축 | 교차 질문 대응 가능 |

## 4. 프로젝트를 통해 배운 것

이 프로젝트에서 가장 큰 교훈은 **"AI의 품질은 AI 모델이 아니라 데이터와 구조에서 결정된다"**는 것입니다.

- 검색 적중률 **+40%p**는 모델 교체가 아니라 **데이터 파싱 3가지 결함 수정**에서 나왔습니다
- 산술 정답률 **+67%p**는 모델 업그레이드가 아니라 **질문 유형별 모델 분리**에서 나왔습니다
- 꼬리질문 일관성 **60%→100%**는 프롬프트 추가가 아니라 **추론 강도를 낮추는 것**에서 나왔습니다
- 환각 방지 **100%**는 AI 기술이 아니라 **AI 교차검증 큐레이션(Decision Trees)**에서 나왔습니다

7개 AI 모델을 218회 호출하여 비교한 결과, 가장 중요한 발견은 "어떤 모델이 좋은가"가 아니라 **"어떤 구조 안에서 모델을 사용하는가"**였습니다.
