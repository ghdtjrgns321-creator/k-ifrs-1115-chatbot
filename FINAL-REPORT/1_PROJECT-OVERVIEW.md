# K-IFRS 1115 회계감사 AI 어시스턴트 — 프로젝트 개요

> **K-IFRS 제1115호(고객과의 계약에서 생기는 수익)** 전문 AI 도구
> 회계감사인이 객관적 팩트를 먼저 확인하고, AI의 구조화된 가이드로 판단을 내릴 수 있도록 설계

---

## 1. 왜 이 프로젝트가 필요한가

### 일반 AI 챗봇의 한계

ChatGPT, Gemini 같은 범용 AI에게 회계 질문을 하면 **그럴듯하지만 틀린 답**을 받을 수 있습니다.
AI 분야에서는 이를 **환각(Hallucination)**이라 부릅니다.

> **환각이란?**
> AI가 학습 데이터에서 패턴을 조합하여 *사실처럼 보이지만 실제 근거가 없는* 답변을 생성하는 현상.
> 회계 맥락에서는 존재하지 않는 기준서 조항을 인용하거나, 실제와 다른 회계처리를 안내하는 것에 해당합니다.

### 회계감사에서 2종오류의 위험성

| 오류 유형 | 정의 | 위험도 |
|-----------|------|--------|
| **1종오류** | 존재하지 않는 오류를 지적함 (과잉 지적) | 낮음 — 추가 검토로 해소 가능 |
| **2종오류** | 실재하는 오류를 놓침 (누락) | **치명적** — 부실감사, 제재, 투자자 피해 |

AI 환각은 곧 **2종오류**(실재하는 회계 쟁점 누락)로 이어집니다.

### 이 시스템의 접근: AI의 자유 추론을 구조적으로 차단

이 프로젝트는 AI에게 자유로운 추론을 허용하지 않습니다. 이를 기술적으로 강제하기 위해 **PydanticAI**라는 프레임워크를 사용합니다.

> **PydanticAI란?**
> AI의 출력 형식을 **코드 수준에서 강제**하는 프레임워크입니다. 일반 챗봇은 AI가 자유형 텍스트를 생성하지만, PydanticAI를 사용하면 AI가 반드시 **정해진 양식**(어떤 분기를 선택했는지, 어떤 문단을 인용했는지)에 맞춰 답변해야 합니다. 양식을 채우지 않으면 답변 자체가 거부됩니다.
>
> 쉽게 말해, AI에게 "자유롭게 써라" 대신 **"이 칸을 채워라"**라고 강제하는 도구입니다.

PydanticAI + AI 교차검증 큐레이션(Decision Trees)을 결합하여:

- AI가 답변할 때 반드시 `인용 문단 번호`와 `선택한 결론 분기`를 채워야 함 — 빈 칸으로는 제출 불가
- 인용이 비어 있으면 **자동으로 반려**하고 AI를 재호출 (최대 2회)
- 사전에 정의된 결론 가이드의 분기만 선택 가능 — AI가 새로운 분기를 **임의로 만들 수 없음**
- AI가 먼저 답하는 것이 아니라, **사용자가 먼저 버튼을 눌러 관련 문단·질의회신·감리사례를 열람**한 뒤 AI에게 질문

## 2. 사용자 경험 흐름

이 시스템은 일반 챗봇처럼 처음부터 AI에게 자유로운 질문을 하는 구조가 아닙니다.
사용자가 먼저 근거 문서를 확인 할 수 있는 **"근거 선행, AI 후행"** 구조입니다.

### 흐름 1: 주제별 문서 열람

```
홈 화면에서 관심 토픽 버튼 클릭 (예: "변동대가", "본인 vs 대리인")
  │
  ▼
토픽 브라우즈 화면: 4개 탭으로 관련 문서를 즉시 열람
  ├─ [본문·BC]     기준서 원문 + 결론도출근거
  ├─ [적용사례]    IE 적용사례 (실무 예시)
  ├─ [질의회신]    QNA 101건 중 해당 토픽 관련
  └─ [감리사례]    FSS·KICPA 감리지적사례
```

AI를 부르기 전에, 해당 토픽의 기준서 본문·적용사례·질의회신·감리사례를 **버튼 하나로 즉시 확인**할 수 있습니다. 30개 토픽(Decision Trees 26개 + 세부 분할 4개)별로 관련 문서가 사전 큐레이션되어 있어 검색할 필요 없이 바로 열람 가능합니다.

### 흐름 2: 질문 → AI 답변 + 근거 문단 Split View

```
자유 질문 입력 (예: "A사가 B사를 통해 소프트웨어를 판매합니다. 본인인가요?")
  │
  ▼
화면이 좌우로 분할됩니다:

┌──────────────────────┬──────────────────────┐
│  [좌측] 근거 문서     │  [우측] AI 답변       │
│                      │                      │
│  AI가 검색해온 문서를  │  근거를 바탕으로      │
│  카테고리별로 표시:    │  구조화된 답변:       │
│                      │                      │
│  ▸ 본문·적용지침      │  [결론]              │
│  ▸ 적용사례 (IE)     │   Case 1: ~이면 본인 │
│  ▸ 질의회신 (QNA)    │   Case 2: ~이면 대리인│
│  ▸ 감리사례          │  [논리적 근거]       │
│  ▸ 참고 추가 문서    │   문단 B37~B38 ...   │
│                      │  + 유사 감리사례 경고 │
│                      │  + 꼬리질문          │
└──────────────────────┴──────────────────────┘
```

**핵심**: AI 답변과 **AI가 가져온 근거 문단**을 한 화면에서 나란히 확인할 수 있습니다.

**정보가 부족한 경우 — 조건부 결론**: AI는 판단에 필요한 정보가 충분하지 않으면 **확정 결론을 내리지 않습니다**. 대신 "Case 1: ~조건이면 본인", "Case 2: ~조건이면 대리인"처럼 **조건별 분기를 나누어 제시**하고, 부족한 정보를 확인하는 꼬리질문을 합니다. 사용자가 추가 정보를 제공하면 그때 분기를 좁혀 결론에 도달합니다.

### 흐름 3: 상세 검색 · 멀티턴 대화

AI 답변 화면에서 **추가 질문**을 입력하면 더 상세한 검색·답변이 가능합니다. 거래 상황 질문의 경우 AI가 **체크리스트 형태의 꼬리질문**으로 상황을 파악합니다:

```
사용자: "A사가 B사를 통해 소프트웨어를 판매합니다. 본인인가요 대리인인가요?"

AI 1턴: "판단에 필요한 핵심 요소를 확인해주세요:
  □ B사가 재화를 통제한 후 고객에게 이전하나요?
  □ B사가 가격결정권을 보유하고 있나요?
  □ B사가 재고위험을 부담하나요?

  현재 정보로는:
  [Case 1] B사가 통제권을 보유 → 본인 (총액 인식)
  [Case 2] A사가 통제권을 보유 → 대리인 (순액 인식)"

사용자: "B사는 가격 결정권이 없고, 재고는 A사에 반품 가능합니다"

AI 2턴: "[결론] B사는 대리인에 해당할 가능성이 높습니다.
  [근거] 문단 B37~B38에 따르면..."
```

## 3. 핵심 기술 개요

| 기술 | 설명 |
|------|------|
| **RAG** (검색 증강 생성) | AI가 답하기 전에 관련 문서를 먼저 DB에서 검색하여 근거로 사용하는 구조 |
| **벡터 검색 + BM25** | 의미 유사도 검색과 키워드 정확 매칭을 RRF로 융합한 하이브리드 검색 |
| **Reranker** | 1차 검색 결과를 Cohere Cross-encoder로 정밀 재평가하여 노이즈 제거 |
| **PydanticAI** | AI 출력 양식을 코드 수준에서 강제 — 인용 필수, 분기 선택 필수, 미충족 시 자동 반려·재시도 |
| **Decision Trees** | 26개 토픽별 체크리스트 + 결론 가이드 — AI가 이 범위 안에서만 답변하도록 제한 |
| **PDR** (부모 문서 검색) | 작은 조각으로 정확히 검색한 뒤, AI에는 원문 전체를 제공하여 맥락 손실 방지 |

## 4. Decision Trees + 환각 방지 5단계 레이어 (핵심 차별점)

이 프로젝트의 가장 독특한 설계는 **MASTER_DECISION_TREES** + **5단계 방어 레이어**의 결합입니다. Decision Trees는 26개 토픽별로 **Gemini·Claude·NotebookLM 3개 AI를 교차검증**하여 작성한 판단 체계이고, 5단계 레이어가 AI를 이 체계 안에 가두는 역할을 합니다.

각 Decision Tree는 **라우팅 키워드 → 체크리스트 → 결론 가이드 → 선례(적용사례·질의회신) → 감리 경고 → 계산 공식** 7개 섹션으로 구성되며, 26개 토픽에 대해 작성되어 있습니다.

### 5단계 방어 레이어: AI를 Decision Tree 안에 가두는 구조

| # | 방어 레이어 | 역할 |
|---|------------|------|
| 1 | **Decision Trees** | 26개 토픽별 체크리스트 + 결론 가이드를 AI 교차검증으로 작성. AI가 답변할 수 있는 범위를 정의 |
| 2 | **PydanticAI 구조화된 출력** | `인용 문단` + `선택 분기`를 반드시 채워야 답변 가능하도록 양식 강제 |
| 3 | **result_validator 자동 재시도** | 인용이 비어 있으면 반려 → AI 재생성 (최대 2회) |
| 4 | **reasoning_guard** | 결론 가이드에 없는 분기를 AI가 임의 생성하는 것을 프롬프트 수준에서 차단 |
| 5 | **선례 직접 주입** | 큐레이션된 적용사례·질의회신·감리사례 원문을 AI에 직접 전달 — 검색 엔진 의존도 축소 |

> 각 토픽의 7개 섹션 구조, 대표 토픽 해설, 토픽 매칭 로직은 `3_DOMAIN-CURATION.md`에서 상세히 다룹니다.

**결과**: AI가 근거 없는 답변을 생성할 수 없는 구조 → **2종오류 방지**

## 5. 검색 아키텍처: 2계층 하이브리드 검색

### Tier 1: 핀포인트 검색

Decision Trees에 사전 배정된 문서 ID로 **MongoDB에서 원문을 직접 조회**합니다. 검색 엔진을 거치지 않으므로 **누락 위험이 0%**입니다.

### Tier 2: 하이브리드 검색

Tier 1이 커버하지 못하는 기준서 본문·적용지침을 보충합니다.

| 검색 방식 | 강점 | 예시 |
|----------|------|------|
| 벡터 검색 | 의미적 유사도 | "수익 인식 시점" ↔ "매출 인식 시기" |
| BM25 키워드 | 정확한 참조 | "문단 B63A", "제35조" |
| RRF 융합 | 두 결과 통합 | 벡터 순위 + 키워드 순위 가중합 |

### Reranker: 2차 검증

Cohere Cross-encoder(`rerank-multilingual-v3.0`)로 1차 검색 결과를 재평가합니다. 핀포인트 문서는 AI 교차검증 큐레이션이므로 Reranker에서 낮은 점수를 받아도 제거되지 않습니다.

## 6. 듀얼트랙 모델 라우팅

7개 AI 모델을 **218회** 호출하여 비교한 결과, **회계 추론 품질과 산술 정확도가 단일 모델로 양립 불가**함을 확인했습니다.

| 모델 | 회계 추론 | 산술 정확도 | 비용 | 역할 |
|------|----------|-----------|------|------|
| **Gemini Flash** (thinking=medium) | **1위** (0.81) | 20~40% | $0.28/150회 | 일반·상황 질문 |
| **gpt-4.1-mini** | 4위 (0.71) | **100%** | $0.04/150회 | 계산 질문 |

```
질문 입력 → analyze_agent가 분석
  ├─ needs_calculation = True   → gpt-4.1-mini (산술 집중)
  ├─ 단순 개념 질문            → Gemini Flash (thinking=low, 빠름)
  └─ 복잡한 판단 질문          → Gemini Flash (thinking=medium, 추론 최고)
```

초기에는 정규식으로 계산 여부를 판단했으나 일관성 33%에 불과 → **AI 판단 방식으로 전환, 42/42 = 100% 정확도**.
